# Configuration Reference

## Environment Variables

All configuration is done via environment variables in `.env`:

| Variable | Default | Description |
|----------|---------|-------------|
| `MOSI_CONCURRENCY` | `3` | Maximum concurrent download jobs |
| `MOSI_PUBLIC_BASE_URL` | - | Public URL for file downloads (required for Telegram) |
| `MOSI_TELEGRAM_BOT_TOKEN` | (empty) | Telegram bot token for notifications |
| `MOSI_TELEGRAM_CHAT_ID` | (empty) | Telegram chat ID for notifications |
| `MOSI_JOB_TIMEOUT` | `1800` | Job timeout in seconds (30 min) |

## Cookie Authentication

For premium or age-restricted content, cookie files are required.

### Supported Sites

| Site | Cookie File | Domains |
|------|-------------|---------|
| YouTube | `cookies/youtube.txt` | youtube.com, youtu.be |
| Twitter/X | `cookies/twitter.txt` | twitter.com, x.com, t.co |
| Instagram | `cookies/instagram.txt` | instagram.com |
| Threads | `cookies/threads.txt` | threads.net |
| Bluesky | `cookies/bluesky.txt` | bsky.app, bsky.social |

### Setting Up Cookies

1. Install the ["Get cookies.txt LOCALLY"](https://chrome.google.com/webstore/detail/get-cookiestxt-locally/) browser extension
2. Log into the desired website
3. Click the extension icon → Export
4. Save the file to `cookies/{site}.txt`

### Cookie File Format

Files must be in Netscape cookie format:
```
# Netscape HTTP Cookie File
# This file was generated by Get cookies.txt LOCALLY
.domain.com	TRUE	/	FALSE	0	cookie_name	cookie_value
```

## Quality Options

When creating a download, specify `quality`:

| Quality | yt-dlp Format Selector | Description |
|---------|------------------------|-------------|
| `best` | (none - default best) | Highest available quality |
| `1080p` | `bv*[height<=1080]+ba/b[height<=1080]/b` | Max 1080p |
| `720p` | `bv*[height<=720]+ba/b[height<=720]/b` | Max 720p |
| `480p` | `bv*[height<=480]+ba/b[height<=480]/b` | Max 480p |
| `audio` | `-x --audio-format mp3` | Audio only as MP3 |

## Download Directory

Downloads are saved to: `~/Downloads/media/`

Files are organized by:
- Source filename preserved
- Job ZIP archives created for multi-file playlists

## Post-Processing

The service applies post-processing based on platform:

### YouTube & General Sites
- Embed thumbnail as cover art
- Embed metadata
- Remux to MP4 if needed

### Apple-Compatible Platforms
For Instagram, Threads, and Bluesky stories:
- Remux to MP4
- Sort for H.264 + AAC codec preference
- Skip thumbnail/metadata embedding (these platforms often have video-only streams that fail metadata embedding)

This behavior is automatic based on URL detection.

## Concurrent Downloads

`MOSI_CONCURRENCY` controls how many downloads run simultaneously. Each worker processes jobs from a shared queue.

Default: 3 concurrent downloads

## Job Cleanup

A systemd timer runs daily to clean up old downloads:

```bash
systemctl status mosi-downloader-cleanup.timer
```

This deletes files in `~/Downloads/media/` older than 7 days.

## Advanced yt-dlp Options

The service uses these yt-dlp options by default:
- `--newline` - Progress output
- `--progress` - Show progress bar
- `--geo-bypass-country US` - Bypass geo-restrictions
- `-P home:{DOWNLOAD_DIR}` - Download location
